Detailed Description

Class scheduling the behavior of the calls executed in tested program.

Class is responsible for creating responses to the messages with calls coming from tested program. It alternates call's normal behavior in order to explore different branches of the tested program. Class is abstract, defines interface for inherited classes, so each of them can use different scheduling strategy. Class is also responsible for creating map of variants used, depending on variant groups specified in configuration file.
